## コンセプト

**「友達から聞いた店を即保存、行きたいときに地図で探す、行ったらメモする」**

グルメ版・行きたいリスト兼振り返り帳。
店は「未訪問（保存済み）」→「訪問済み（メモ済み）」の2状態を持つ。

## コアUXフロー

### 保存フロー（第一優先）
```
友達から店を聞く
  → アプリ起動 → 店名を検索
  → 候補タップ → 「保存」1タップ  ← 30秒以内が目標
  → メモ・タグは後で追加可
```

### 探すフロー（第一優先）
```
「今日何食べようかな」
  → マップ起動 → 現在地表示
  → 🔴未訪問 / 🟢訪問済みのピン
  → 上部フィルターチップで絞り込み
  → ピンタップ → 店舗カード
```

### 訪問後フロー（第二優先）
```
食べた直後 or 電車の中
  → 店舗カードを開く
  → ★評価・メモ・写真・訪問日を追加
  → 未訪問 → 訪問済みに変更
```

## MVP機能スコープ

### ✅ Phase 1で作るもの

| 機能 | 詳細 |
|------|------|
| 店舗検索・1タップ保存 | Google Places API連携（日本語店名対応） |
| マップ表示 | 🔴未訪問 / 🟢訪問済みピン、現在地、クラスタリング |
| マップフィルター | 上部ピル形チップ（状態・タグ） |
| リスト表示 | 登録日新しい順、タグ・状態フィルター付き |

## 技術スタック

| 領域 | 採用技術 | 理由 |
|------|---------|------|
| フロントエンド | Next.js (React) | PWA対応、SEO、Vercel連携 |
| 地図・検索 | Google Maps API + Places API | 精度◎、日本の飲食店データが最充実 |
| バックエンド/DB | Supabase (PostgreSQL) | 認証・DB・ストレージ一体、無料枠あり |
| 認証 | Supabase Auth（Googleログイン） | 1タップ、パスワード不要 |
| 画像ストレージ | Supabase Storage | 無料枠1GB（圧縮後500KB × 2,000枚分） |
| ホスティング | Vercel | Next.jsと完全連携 |
